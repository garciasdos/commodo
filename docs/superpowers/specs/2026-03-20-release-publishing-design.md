# Release & Publishing Flow Design

## Overview

Automated release pipeline for commodo: GitHub Releases with cross-platform binaries, checksums, and auto-published Homebrew formula.

## Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Trigger | GitHub Release UI (`release: published`) | Manual control over when to release |
| Tag/version | Fully manual, typed in GH UI | Simple, no validation overhead |
| Build tool | GoReleaser | Industry standard for Go CLIs, handles builds + checksums + Homebrew tap in one config |
| Homebrew distribution | `garciasdos/homebrew-tap` | Required by Homebrew for third-party formulas |
| Checksums | SHA256 `checksums.txt` | Standard practice, used by Homebrew formula for verification |
| License | MIT | Existing project license |

## Architecture

### Release Workflow

**Trigger:** `release: [published]` event — fired when a release is created via the GitHub UI.

**File:** `.github/workflows/release.yml`

```yaml
# Pseudostructure
on:
  release:
    types: [published]

permissions:
  contents: write

jobs:
  release:
    runs-on: ubuntu-latest
    steps:
      - checkout (with fetch-depth: 0 for goreleaser changelog)
      - setup-go (go-version-file: go.mod)
      - goreleaser/goreleaser-action (with GITHUB_TOKEN and HOMEBREW_TAP_TOKEN)
```

**Key details:**
- `fetch-depth: 0` is required so GoReleaser can generate changelogs from git history
- `contents: write` permission to upload release assets
- `HOMEBREW_TAP_TOKEN` secret (GitHub PAT with `repo` scope) for pushing formula to tap repo
- `GITHUB_TOKEN` is used by GoReleaser to upload assets to the release

### GoReleaser Configuration

**File:** `.goreleaser.yaml` (repo root)

**Builds:**
- Single build entry for `github.com/garciasdos/commodo`
- Targets: `linux/amd64`, `linux/arm64`, `darwin/amd64`, `darwin/arm64`
- Ldflags: `-s -w -X main.version={{.Version}}` (injects version from git tag, strips debug info)

**Archives:**
- Format: `tar.gz`
- Name template: `commodo_{{.Version}}_{{.Os}}_{{.Arch}}`
- Contents: binary only (no extra docs)

**Checksum:**
- Algorithm: SHA256
- Output: `checksums.txt`

**Changelog:**
- Auto-generated from commits since last tag
- Grouped by conventional commit type (feat, fix, chore, etc.)

**Brews:**
- Tap: `garciasdos/homebrew-tap`
- Formula name: `commodo`
- Description: "CLI tool that generates conventional commit messages from staged git diffs using LLMs"
- Homepage: `https://github.com/garciasdos/commodo`
- License: MIT
- Install: copies binary to `bin`
- Token: `{{ .Env.HOMEBREW_TAP_TOKEN }}`

### Homebrew Tap Repository

**Repo:** `garciasdos/homebrew-tap` (public, already created)

**Structure:**
```
homebrew-tap/
  Formula/
    commodo.rb    # Auto-generated by GoReleaser on each release
```

The formula is fully managed by GoReleaser. It contains download URLs for each OS/arch, SHA256 checksums, and install instructions.

**User install:**
```bash
brew install garciasdos/tap/commodo
```

## End-to-End Flow

1. Development happens on branches, PRs merged to `main` (existing CI runs vet/test/build)
2. When ready to release: GitHub → Releases → Create new release
3. Type a new tag (e.g. `v1.0.0`), target `main`, write release notes, publish
4. `release.yml` triggers → GoReleaser:
   - Cross-compiles 4 binaries (linux/darwin x amd64/arm64)
   - Creates `.tar.gz` archives
   - Generates `checksums.txt` (SHA256)
   - Uploads archives + checksums to the GitHub Release
   - Pushes updated `Formula/commodo.rb` to `garciasdos/homebrew-tap`
5. Users run `brew install garciasdos/tap/commodo` or `brew upgrade commodo`

## Files Changed

| File | Action | Purpose |
|------|--------|---------|
| `.goreleaser.yaml` | New | GoReleaser configuration |
| `.github/workflows/release.yml` | New | Release workflow |

## Manual One-Time Setup

1. Create a GitHub PAT (classic) with `repo` scope
2. Add it as `HOMEBREW_TAP_TOKEN` secret in the `commodo` repo settings (Settings → Secrets → Actions)

## What Stays Unchanged

- Existing `ci.yml` workflow (vet, test, build matrix for PRs)
- `main.go` and all Go code
- `update-models.yml` workflow
